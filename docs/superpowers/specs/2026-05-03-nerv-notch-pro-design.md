# NERV Notch Pro 设计规格

日期：2026-05-03

## 目标

NERV Notch Pro 是一个原生 macOS 刘海状态岛应用，用于在 MacBook 刘海区域展示当前电脑状态参数。首版聚焦 CPU、内存、网络负载三类真实数据，并在刘海 Island 展开后模拟《新世纪福音战士》中 NERV 本部指挥室与 MAGI 系统的控制台体验。

项目定位为本地个人使用的粉丝向应用。首版允许直接使用 NERV、MAGI、MELCHIOR、BALTHASAR、CASPER 等命名和语境，但不内置受版权保护的图片素材。后续如果需要开源发布或商业分发，应增加原创命名与无版权素材模式。

## 首版范围

首版实现：

- 原生 macOS App。
- 真实刘海机型优先，非刘海 Mac 使用模拟刘海。
- 点击刘海展开，点击外部收起。
- 鼠标持续悬停刘海区域 2 秒后展开，移出展开面板焦点区域后收起。
- 三个 MAGI 独立决策窗口：CPU、内存、网络。
- 一个 Central Dogma 总控裁决区。
- CPU、内存、网络真实采集。
- 采集失败时局部降级，不影响其他窗口。
- 基础设置：目标屏幕、模拟刘海、悬停延迟、采样频率。

首版不实现：

- GPU 或显存采集。
- 进程级详细排行。
- 历史记录持久化。
- 通知中心告警推送。
- 云同步或远程监控。
- 发布用版权规避模式。

## 技术路线

采用原生 Swift/macOS App，底层参考 `/Users/aarondai/Develop/Codebase/vibe-notch` 的刘海基座思路，但在本项目中独立实现。

架构分层：

- `App`：应用生命周期、菜单栏入口、启动配置、权限提示。
- `Notch`：刘海窗口、屏幕选择、几何计算、点击和悬停交互、外部点击收起。
- `Telemetry`：CPU、内存、网络采集。
- `MagiSystem`：把系统指标转换为三路 MAGI 决策和总控裁决。
- `NervConsoleUI`：SwiftUI 控制台组件、MAGI 窗口、扫描线、网格、警戒边框、状态文本。
- `Settings`：目标屏幕、模拟刘海、悬停延迟、采样频率等用户配置。

数据流保持单向：

`TelemetrySampler` 定时生成 `SystemSnapshot`，`MagiDecisionEngine` 将快照转换为 `MagiDecisionState`，`NotchViewModel` 暴露 SwiftUI 渲染所需状态。窗口行为、数据采集、决策逻辑和 UI 渲染保持边界清晰，便于测试和替换。

## 刘海窗口与交互

应用为目标屏幕创建顶部全宽透明 `NSPanel`。窗口常驻所有 Space，浮在菜单栏之上，不参与普通窗口切换。关闭状态下窗口允许鼠标事件穿透，避免影响菜单栏和其他应用；展开状态下窗口接管鼠标事件，使 MAGI 面板内控件可点击。

窗口基座参考以下原则：

- 使用无边框、非激活、透明 `NSPanel`。
- 关闭状态 `ignoresMouseEvents = true`。
- 展开状态 `ignoresMouseEvents = false`。
- 屏幕变化、分辨率变化、多屏切换、休眠唤醒后重新计算几何并重建窗口。
- 真实刘海机型使用物理刘海区域；非刘海设备使用居中模拟 Island。

交互状态：

- `closed`：显示贴合刘海的小型 NERV 状态条。
- `hoverArming`：鼠标进入刘海命中区域后启动 2 秒计时。
- `opened`：点击刘海或悬停满 2 秒后展开 MAGI 面板。
- `closing`：点击外部或鼠标离开展开面板焦点区后执行收起动画。

交互规则：

- 点击刘海区域立即展开。
- 点击展开面板外部立即收起。
- 悬停刘海区域持续 2 秒后展开。
- 鼠标离开展开面板区域后自动收起，并保留 150-250ms 缓冲，避免边缘抖动。
- 展开时不主动抢占前台应用焦点，但面板本身可以接收交互。

展开面板首版固定为居中布局，建议宽度 720-860px，高度 360-460px。面板从刘海向下展开，视觉上像 MAGI 判断层被解锁，而不是普通弹窗。

## 系统数据采集

采样默认每 1 秒执行一次。UI 动效可以独立运行，但数值更新保持采样节奏，避免闪烁和不必要的资源占用。

### CPU

CPU 采集使用 `host_processor_info` 读取各核心 tick，通过两次采样差值计算整体 CPU 使用率。首版显示整体 CPU 百分比、核心数、用户/系统/空闲比例。首版不采集最热进程，避免额外权限和复杂度。

### 内存

内存采集使用 `host_statistics64` 获取 active、inactive、wired、free、compressed 等页面数据，并使用 `ProcessInfo.physicalMemory` 获取物理内存总量。首版输出已用比例、可用内存、压缩内存和内存压力等级。Swap 信息作为后续扩展，不作为首版核心判断条件。

### 网络

网络采集使用 `getifaddrs` 读取活跃网络接口输入/输出字节计数，通过两次采样差值计算上行和下行速率。默认排除 `lo0`，统计实际通信接口。无活跃接口时，CASPER 显示 `COMM LINK IDLE`，不视为错误。

## MAGI 决策模型

三个 MAGI 窗口分别对应三类系统指标：

- `MELCHIOR-01 / CPU LOAD`：处理器负载、核心压力、计算状态。
- `BALTHASAR-02 / MEMORY`：内存占用、可用内存、稳定性压力。
- `CASPER-03 / NETWORK`：上下行速率、接口状态、通信链路状态。

每个窗口输出：

- 主数值。
- 最近 30-60 秒的小型条形图或波形图。
- 状态标签，如 `NORMAL`、`HIGH LOAD`、`CRITICAL`、`SIGNAL LOST`。
- 一行决策语句，如 `LOAD ACCEPTABLE`、`MEMORY PRESSURE RISING`、`COMM LINK ACTIVE`。

总控裁决区命名为 `CENTRAL DOGMA JUDGEMENT`，综合三路决策输出整体状态：

- `SYNCHRONIZED`：三路状态正常。
- `ELEVATED ALERT`：至少一路进入高负载。
- `EMERGENCY MODE`：至少一路进入临界状态。
- `PARTIAL SYNC`：至少一路数据不可用，但系统仍可局部判断。

状态升级需要连续 3 次采样达到高负载或临界条件，避免瞬时峰值误报。状态恢复需要连续 5 秒处于正常区间，保证总控裁决具备稳定感。

## NERV/MAGI 视觉设计

首版视觉目标是 NERV 本部控制台与 MAGI 独立决策窗口，而不是现代 macOS 监控卡片。

视觉语言：

- 黑色和深红为主色。
- 警戒橙、白色细线、终端绿作为辅助色。
- 硬边框、切角、网格、扫描线、编号标签、状态戳、高密度小字。
- 避免圆角卡片、毛玻璃、柔软阴影和普通仪表盘风格。

展开结构：

- 顶部：`NERV HQ / MAGI SYS` 标识区，显示同步状态、采样时间和接口状态。
- 中部：三列 MAGI 独立决策窗口。
- 底部：`CENTRAL DOGMA JUDGEMENT` 总控裁决区。
- 背景：细网格、弱扫描线、角标、红色警戒边框和少量随机闪烁的数据片段。

动效原则：

- 展开/收起像机械面板解锁，高度、透明度、扫描线同步变化。
- 数值变化使用短暂闪烁或字符跳变。
- 高负载时红色警戒边框增强，窗口标题条短促闪烁。
- 动效必须轻量，不能让监控应用本身成为明显 CPU 消耗来源。

## 设置

首版设置项：

- 目标屏幕选择。
- 模拟刘海开关。
- 悬停展开延迟，默认 2 秒，可选 1-4 秒。
- 采样频率，默认 1 秒，可选 1 秒、2 秒、5 秒。
- 启动时自动运行入口可以预留；如果实现成本影响首版，则延后到第二阶段。
- 粉丝向模式默认开启；原创命名模式延后到发布准备阶段。

## 错误处理与降级

错误处理必须局部化：

- 无法识别刘海区域时使用模拟刘海，并显示 `SIMULATED NOTCH ACTIVE`。
- CPU 采集失败时，MELCHIOR 显示 `DATA UNAVAILABLE`。
- 内存采集失败时，BALTHASAR 显示 `DATA UNAVAILABLE`。
- 网络无活跃接口时，CASPER 显示 `COMM LINK IDLE`。
- 网络采集失败时，CASPER 显示 `SIGNAL LOST`。
- 任一采集器失败不影响其他窗口继续刷新。
- 总控裁决在部分数据不可用时输出 `PARTIAL SYNC`。

应用不得因采集异常导致 UI 崩溃。采集服务应捕获错误并输出结构化降级状态。

## 测试策略

单元测试：

- `NotchGeometry`：真实刘海、模拟刘海、多屏坐标、展开区域命中、外部点击判断。
- `Telemetry`：CPU 差值计算、内存比例计算、网络速率差值计算、无活跃接口处理。
- `MagiDecisionEngine`：正常、高负载、临界、部分数据缺失、状态升级和恢复滞后。
- `NotchViewModel`：点击展开、悬停 2 秒展开、移出收起、点击外部收起。

手动验证：

- 刘海机型真实显示。
- 非刘海设备模拟 Island。
- 多显示器目标屏幕切换。
- Space 切换。
- 全屏 App 场景。
- 休眠唤醒后窗口位置恢复。
- 点击穿透和展开后控件交互。

性能验收：

- 应用空闲常驻时 CPU 占用保持很低。
- 采样频率不会造成可感知卡顿。
- 展开动效和扫描线不能显著增加系统负载。

## 验收标准

首版完成时应满足：

- 应用可启动并在目标屏幕刘海区域显示 NERV 小型状态条。
- 点击刘海可展开 MAGI 面板，点击外部可收起。
- 悬停刘海 2 秒可展开，移出面板区域可收起。
- CPU、内存、网络数据真实刷新。
- 三个 MAGI 窗口和总控裁决区可根据数据变化更新状态。
- 采集失败时对应窗口降级，应用不崩溃。
- 非刘海设备可使用模拟刘海完成主要体验。
- UI 明确呈现 NERV/MAGI 指挥室控制台风格。
